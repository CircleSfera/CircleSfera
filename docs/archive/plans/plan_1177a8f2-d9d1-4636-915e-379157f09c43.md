# Implementation Plan - Fix Avatar Upload

The avatar upload fails because `class-validator`'s `@IsUrl` validation might be too strict for local development URLs (e.g., `http://localhost:3000/...`) or the frontend is sending a path instead of a full URL.

## Diagnosis

- Frontend uploads file -> Backend returns URL (e.g. `http://localhost:3000/uploads/uuid.jpg`).
- Frontend sends `PUT /api/v1/profiles/me` with `{ avatar: "http://localhost:3000/uploads/uuid.jpg" }`.
- Backend `UpdateProfileDto` has `@IsUrl()` decorator.
- `class-validator` default `IsUrl` options might reject `localhost`.

## Proposed Changes

### Backend

#### [MODIFY] [update-profile.dto.ts](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/backend-api/src/profiles/dto/update-profile.dto.ts)

- Change `@IsUrl()` to `@IsString()` or configure `@IsUrl({ require_tld: false })` to allow localhost URLs.
- Given we are in dev environment, relaxing to `@IsString()` or optional URL validation is safer to unblock the user.
