# Implementation Plan - Direct Messaging (Chat)

The goal is to finalize the Direct Messaging (Chat) feature by cleaning up the backend implementation and ensuring robust real-time communication. The frontend already has the necessary logic in `ChatWindow.tsx`, but the backend services need refactoring to remove technical debt (`any` casts, `@ts-nocheck`) and ensure type safety.

## User Review Required

> [!NOTE]
> We will stick to the **HTTP-First** approach for sending messages. The client posts to `/api/v1/chat/messages`, and the backend emits the WebSocket event `receiveMessage`. The existing `sendMessage` socket event in `ChatGateway` will be removed or marked as deprecated to avoid confusion.
