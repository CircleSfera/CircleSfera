# Story Improvements & Highlights Fix

## Goal Description

Enhance the Stories feature by adding the ability to delete stories and see who viewed them. Fix a bug where Story Highlights are not saving.

## User Review Required

None so far.

## Proposed Changes

### Backend (NestJS)

#### [Highlights]

- **Fix Controller Path**: Change `@Controller('highlights')` to `@Controller('api/v1/highlights')` in `HighlightsController`. This matches the frontend API call.

#### [Stories]

- **DELETE /api/v1/stories/:id**: Add `@Delete(':id')` to `StoriesController` calling `storiesService.delete`.
- **POST /api/v1/stories/:id/view**: Add `@Post(':id/view')` calling a new `storiesService.view` method.
  - `view(storyId, userId)`: Upsert `StoryView` (db level unique constraint handles duplicates, or check first).
- **GET /api/v1/stories/:id/views**: Add `@Get(':id/views')` calling `storiesService.getViews`.
  - `getViews(storyId)`: Return users who viewed the story (include profile).

### Frontend (React)

#### [Components]

- **StoryViewer**:
  - **View Tracking**:
    - `useEffect`: Call `storiesApi.markViewed(story.id)` when current story changes.
  - **Owner Controls**:
    - If `story.userId === currentUser.id`:
      - Show "Delete" button. On click -> confirm -> `storiesApi.delete` -> close viewer/refresh.
      - Show "Eye" icon with count. On click -> open modal/popover with viewer list (`storiesApi.getViews`).

#### [Services]

- **index.ts (storiesApi)**:
  - Update `markViewed` path to `/api/v1/stories/${id}/view`.
  - Update `getViews` path to `/api/v1/stories/${id}/views`.
  - Add `delete: (id: string) => apiClient.delete('/api/v1/stories/' + id)`.

## UI Improvements Plan

1. **Visualization**: Add a background layer with a blurred version of the story media to fill empty space, creating an immersive experience.
2. **Delete Dialog**: Replace `window.confirm` with a custom styled modal that matches the app's dark theme.
3. **Viewers List**: Refine the viewers sheet with better rounded corners, a handle bar, and smoother entrance animation.

## Video Creation Fixes (Stories & Posts)

1. **Carousel Preview**: Update `Carousel.tsx` to render `<video>` tags when `type === 'video'`. This fixes preview for both Story and Post creation.
2. **Upload Limit**: Increase the `MaxFileSizeValidator` in `UploadsController` from 10MB to 50MB to support reasonable video sizes.
3. **Error Handling**: Catch specific errors in `CreatePostModal` (like 413 Payload Too Large) and show user-friendly messages.

   - Update `PhotoEditor.tsx` to detect if the file is a video.

   - Ensure thumbnails also render `<video>` (muted) so users can preview filters on the video.
2. **UI/UX Polish**:
   - **User Avatar**: Fetch `profile` from `useAuthStore` to display the actual user's avatar in the final caption step instead of a placeholder gradient.
