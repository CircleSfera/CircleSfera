# Plan de Implementación: Funcionalidades Críticas

Este plan aborda las carencias más importantes identificadas en la auditoría, enfocándose primero en la seguridad y la arquitectura de identidad, seguido por la integración real de IA.

## User Review Required

> [!IMPORTANT]
> La implementación de la recuperación de contraseña y verificación de email requiere un **Servicio de Email** (NodeMailer con SMTP o proveedores como SendGrid/AWS SES). Necesito confirmar si tienes preferencia por algún proveedor o si usamos una configuración SMTP genérica para desarrollo.

## Verification Plan

### Automated Tests

- `npm run test` en el backend para verificar que los tokens se generan y validan correctamente.
- Mock de `EmailService` en tests para no enviar correos reales durante las pruebas.

### Manual Verification

1. Intentar hacer login con una cuenta no verificada (si se activa el bloqueo).
2. Solicitar cambio de contraseña y verificar la recepción del email (usando herramientas como Mailtrap).
3. Resetear contraseña y entrar con la nueva.
4. Realizar una búsqueda "Explorar" y verificar que los resultados de IA ya no son aleatorios.
