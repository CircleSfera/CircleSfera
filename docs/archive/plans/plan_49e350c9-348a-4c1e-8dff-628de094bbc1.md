# Implementation Plan - Fix Story Avatar Hover Effect

The current hover effect on the user's story avatar (when a story is uploaded) applies `opacity-80` to the container div that has a black background. This container is used to mask the center of the gradient div behind it, creating a pseudo-border. Making it transparent reveals the gradient behind the avatar, which results in a muddy/incorrect visual.

## User Review Required

> [!NOTE]
> No breaking changes. This is a visual fix only.

## Proposed Changes

### Frontend App

#### [MODIFY] [StoryList.tsx](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/frontend-app/src/components/StoryList.tsx)

- Update the hover logic for the "Your Story" button (lines 73-82).
- Remove `group-hover:opacity-80` from the black background div.
- Add `group-hover:opacity-80` (or similar) to a wrapper around `UserAvatar` or pass it via `className` to `UserAvatar`.
- Consider adding a scale effect for a better "premium" feel as requested in general design guidelines.

**Specific Change:**
Change this:

```tsx
<div className="bg-black rounded-full p-[1.5px] relative group-hover:opacity-80 transition-opacity">
  <UserAvatar ... />
</div>
```

To something like:

```tsx
<div className="bg-black rounded-full p-[1.5px] relative overflow-hidden">
  <div className="transition-opacity group-hover:opacity-80">
    <UserAvatar ... />
  </div>
</div>
```

Or simply remove the opacity change if it feels unnecessary, but a slight interaction feedback is good.

I will also apply the same fix to the "other users' stories" section (lines 136-145) to maintain consistency.
