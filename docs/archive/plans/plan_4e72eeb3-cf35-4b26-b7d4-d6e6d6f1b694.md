# CSRF and Socket Connection Stabilization

Resolve the `invalid csrf token` errors and `Socket connection failed` issues by implementing a robust CSRF management system and stabilizing session handshakes.

## Technical Problem

The `access_token` rotates every 15 minutes. Because `doubleCsrf` was using the full token string as the `getSessionIdentifier`, every rotation invalidated all existing CSRF tokens, causing a "CSRF Ping-Pong" effect:

1. Access token expires.
2. Background requests fail with 403 (Invalid CSRF) before they can trigger 401 (Refresh).
3. CSRF token mismatch deadlocks until the session is manually refreshed or a retry loop occurs.

## Proposed Changes

### [Component] Frontend API & Auth

#### [MODIFY] [api.ts](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/circlesfera-frontend/src/services/api.ts)

- **Race Condition Protection**: Use a shared promise (`csrfTokenPromise`) to ensure multiple concurrent requests only trigger one CSRF fetch.
- **Session Transitions**: Automatically clear and refresh tokens after `/auth/login`, `/auth/register`, and `/auth/refresh`.
- **403 Resilience**: Automatically retry requests once if a CSRF failure occurs.

#### [MODIFY] [socketStore.ts](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/circlesfera-frontend/src/stores/socketStore.ts)

- **Auth Trigger**: Attempt a session refresh if a socket connection fails with "No token found" or JWT-related errors.
- **Backoff & Limit**: Limit refresh attempts to 3 to prevent infinite loops.

### [Component] Backend Configuration

#### [MODIFY] [csrf.config.ts](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/circlesfera-backend/src/common/config/csrf.config.ts)

- **Stable Identifier**: Change `getSessionIdentifier` to extract the `sub` (userId) from either the `access_token` or `refresh_token`. This makes the CSRF token stable for 7 days (the lifespan of the refresh token) instead of 15 minutes.

#### [MODIFY] [main.ts](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/circlesfera-backend/src/main.ts)

- **Exclusions**: Explicitly exclude `/api/v1/auth/refresh` from CSRF protection to allow session recovery during token rotation failures.

#### [MODIFY] [all-exceptions.filter.ts](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/circlesfera-backend/src/common/filters/all-exceptions.filter.ts)

- **Detailed Logging**: Include HTTP Method and URL in unhandled exception logs (especially for CSRF errors) to facilitate easier debugging.
