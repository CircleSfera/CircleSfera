# Nested Comments Implementation Plan

## Goal Description

Implement nested comments (replies) feature to allow users to reply to specific comments. This involves updating the data model, backend API, and frontend UI to support hierarchical comment structures.

## Proposed Changes

### Backend (PostgreSQL + Prisma + NestJS)

#### [MODIFY] [schema.prisma](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/backend-api/prisma/schema.prisma)

- Add self-relation to `Comment` model: `parentId` field and `replies` relation.

#### [MODIFY] [comments.service.ts](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/backend-api/src/comments/comments.service.ts)

- Update `create` method to accept `parentId` in DTO.
- Update `findAll` logic. We will fetch flat list and reconstruct tree in frontend, or rely on Prisma `include` if depth is limited (fetching flat is safer for arbitrary depth, but for now we might stick to 1 level deep).
- **Decision**: We will support 1 level of nesting in the UI for simplicity (like Instagram often does essentially 1 level of visual indentation or "view replies"). For full threading, we'd need recursive fetching or CTEs. Let's start with supporting the data model for arbitrary depth, but maybe just fetch 1 level for now or fetch all flat for specific post and weave them. Fetching all comments for a post is standard.

#### [MODIFY] [comments.controller.ts](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/backend-api/src/comments/comments.controller.ts)

- Update `create` DTO to include optional `parentId`.
