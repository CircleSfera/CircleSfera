# Fix Content Upload Preview and Layout Issues

The user is experiencing layout issues during content upload (Post, Story, Frames), specifically unwanted scrolling and poor preview visualization. This is caused by the `EditStep` component enforcing a fixed width and aspect ratio for the preview container, which, when combined with the modal's layout, exceeds the vertical viewport space on many screens.

## User Review Required

> [!NOTE]
> I will be prioritizing "fitting the content" over "fixed size previews". This means the preview image might appear smaller on some screens to ensure the entire UI (including buttons and thumbnails) is visible without scrolling.

## Proposed Changes

### Frontend Components

#### [EditStep.tsx](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/circlesfera-frontend/src/components/create-post/EditStep.tsx)

- **Layout Refactor**: Change the preview container logic. Instead of `w-full` + `aspect-ratio` (which drives height based on width), use a flex-grow approach where the image container takes available space and the image/carousel inside uses `object-fit: contain` (or equivalent layout technique) to fit within that space without pushing boundaries.
- **Remove Fixed Constraints**: Remove `max-w-[320px]` / `max-w-[260px]` hard constraints that might conflict with height limits. Let the parent flex container dictate the size.

#### [CreatePostModal.tsx](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/circlesfera-frontend/src/components/CreatePostModal.tsx)

- **Modal Height Logic**: Ensure the modal's `max-h` logic works in tandem with the inner `flex-col` to prevent overflow.
- **Scroll Management**: Verify `overflow-y-auto` is only active when absolutely necessary (e.g. for long lists), but for the main "Edit" view, we want to avoid it if possible.

#### [PhotoEditor.tsx](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/circlesfera-frontend/src/components/PhotoEditor.tsx)

- **Preview Sizing**: Ensure the preview container uses `flex-1` and `min-h-0` to properly shrink.
- **Image Fit**: Use `object-contain` for the preview image to ensure the entire image is visible for editing, rather than `object-cover` which crops it.
- **Layout**: Ensure the editor controls (filters, adjustments) do not overlay the image or get pushed off-screen.

#### [EditStep.tsx](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/circlesfera-frontend/src/components/create-post/EditStep.tsx)

- **Interaction**: Make the edit interaction more obvious, perhaps by adding an "Edit" button overlay on the image.

### [Frames.tsx](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/circlesfera-frontend/src/pages/Frames.tsx)

- **Background**: Remove `bg-black` from the main container and empty/loading states. Use `bg-transparent` or let the global background show through to match the project's gradient theme.
