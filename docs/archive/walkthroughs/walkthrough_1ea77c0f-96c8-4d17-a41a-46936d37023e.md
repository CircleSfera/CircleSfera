# Walkthrough: Universal PPV Monetization & Verification System (v2)

Hemos completado la implementación y refinamiento del sistema de monetización Universal PPV y la taxonomía de niveles de verificación.

## 1. Monetización Universal PPV
- **Funcionalidad**: Los creadores pueden ahora monetizar cualquier publicación o historia con precios PPV ajustables.
- **Visualización**: El contenido premium aparece bloqueado para usuarios sin compra, con indicadores claros de precio.

## 2. Niveles de Verificación y Roles
- **Taxonomía**: Implementados niveles `BASIC`, `VERIFIED`, `BUSINESS` y `ELITE`.
- **Identidad**: Badges visuales inyectados en Feed, Historias y Cabecera de Perfil con colores distintivos.

## 3. Gestión Administrativa (Panel Admin)
- **UserVerificationTab**: Nueva interfaz para gestionar el estatus de confianza de los usuarios.
- **Edición en Tiempo Real**: Cambios instantáneos de nivel de verificación y tipo de cuenta (`PERSONAL`, `CREATOR`, `BUSINESS`).
- **Seguridad**: Filtrado estricto por rol de Administrador y registro en el `Audit Log`.

## 4. Refinamiento Técnico y Calidad de Código
- **Tipado Seguro**: Eliminados todos los "unsafe any" en el backend y frontend.
- **Sincronización Prisma**: Implementadas interfaces de casting seguro (`UserWithVerification`) para manejar los nuevos campos del esquema.
- **ESLint Compliance**: Formateo y lógica de tipos validados bajo los estándares profesionales del proyecto.

---
**Resultado**: El sistema está completamente integrado, tipado y listo para ser probado en el entorno de desarrollo local antes de su despliegue.

## 5. Guía de Ejecución en Docker Local

Para levantar el proyecto completo localmente, sigue estos pasos:

### Paso 1: Levantar los contenedores
Desde la raíz del proyecto, ejecuta:
```bash
docker compose up --build -d
```

### Paso 2: Inicializar la Base de Datos
Una vez que los contenedores estén corriendo (puedes verificar con `docker ps`), inicializa el esquema y los datos de prueba:
```bash
# Aplicar migraciones de Prisma
docker exec -it CircleSfera-Backend npx prisma migrate deploy

# Poblar con datos semilla (usuarios, posts, etc.)
docker exec -it CircleSfera-Backend npm run prisma:seed
```

### Paso 3: Acceder a las aplicaciones
- **Frontend App**: [http://localhost:8080](http://localhost:8080)
- **Landing Page**: [http://localhost:8081](http://localhost:8081)
- **API Documentation**: [http://localhost:3000/api/docs](http://localhost:3000/api/docs)

> [!IMPORTANT]
> El archivo `.env` de la raíz ha sido sincronizado con las credenciales de producción pero adaptado para la red interna de Docker (`DB_HOST=postgres`). Si necesitas ejecutar el backend fuera de Docker, deberás cambiar `DB_HOST` a `localhost`.
