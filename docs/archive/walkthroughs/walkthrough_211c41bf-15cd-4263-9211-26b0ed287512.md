# Walkthrough - Socket Authentication Fix

The "No token found" error occurring during WebSocket handshakes has been successfully resolved. The root cause was inconsistent token extraction from cookies in the socket environment, which has now been stabilized and verified.

## Changes Made

### Backend

- **[app.gateway.ts](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/circlesfera-backend/src/socket/app.gateway.ts)**:
  - Stabilized `extractToken` to reliably parse cookies from the socket handshake.
  - Improved error handling and logging for failed connection attempts.
  - Removed diagnostic file logging (`socket-debug.log`) after verifying successful handshakes.

### Frontend

- **[socketStore.ts](file:///Users/ShadyFeliu/Desktop/Projects/CircleSfera/circlesfera-frontend/src/stores/socketStore.ts)**:
  - Added a more robust `connect_error` handler that triggers a session refresh on authentication/CSRF failures.
  - Verified `withCredentials: true` ensures cookies are sent correctly even in polling-to-websocket transitions.
