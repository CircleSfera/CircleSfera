# Dockerization Walkthrough

## Overview

This document outlines the full-stack containerization of CircleSfera, enabling consistent local development and production deployment.

### 1. Root Orchestration (`/docker-compose.yml`)

- **New**: Created a master `docker-compose.yml` (modern spec, no version field) to orchestrate 4 services:
  - `postgres`: Database (v16).
  - `redis`: Caching/Queue (v7).
  - `backend`: NestJS API (Port 3000).
  - `frontend`: React + Nginx (Port 8080).

### 2. Frontend Configuration

- **Updated** `circlesfera-frontend/Dockerfile`: Added step to copy `nginx.conf` to the container.
- **New** `circlesfera-frontend/.dockerignore`: Prevents local `node_modules` from overwriting container dependencies during build.

### 3. Backend Configuration

- **Updated** `circlesfera-backend/Dockerfile`: Added `apk add --no-cache openssl` to the production stage. This is **critical** for Prisma Client to function correctly on Alpine Linux.

### 4. Security & Optimization (Audit)

- **Backend**: Switched to non-root user (`USER node`) and ensured correct file permissions (`--chown=node:node`).
- **Frontend**: Created `.dockerignore` to prevent local `node_modules` from contaminating the build.
- **Compose**: Renamed legacy `circlesfera-backend/docker-compose.yml` to `docker-compose.dev.yml` to avoid confusion.
